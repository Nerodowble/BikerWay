Este documento contextualiza o produto, define o escopo, as regras de negócio e a lógica matemática que a IA precisará programar.1. Visão Geral do ProdutoO BikerWay é um aplicativo de navegação assistida e inteligência preditiva projetado exclusivamente para motociclistas. Diferente de GPS comuns, o sistema calcula rotas baseando-se no perfil de pilotagem (curvas, pavimento) e gerencia ativamente a autonomia de combustível do veículo em tempo real, eliminando a necessidade de interações complexas com a tela durante a condução.2. Personas e Casos de Uso CorePiloto Urbano / Scooter (Ex: Honda PCX): Busca rotas expressas, pavimentadas, sem vias de terra, com foco em alertas automáticos de reabastecimento devido ao tanque pequeno.Piloto de Turismo / Esportiva (Ex: Kawasaki Ninja): Busca rotas cênicas, com alta sinuosidade (curvas), asfalto perfeito e postos com combustível premium (alta octanagem).Piloto de Aventura / Trail (Ex: BMW GS): Permite rotas com trechos de terra batida (unpaved) e necessita de pontos de apoio mecânico/borracharias isoladas.3. Especificação de Funcionalidades (Features)RF01: Pré-Cadastro de Veículo e Perfil de ConsumoO usuário deve cadastrar: Marca, Modelo, Capacidade do Tanque (L) e Consumo Médio Estimado (km/L).O sistema deve calcular a Autonomia Teórica Máxima ($A_{max} = \text{Capacidade} \times \text{Consumo}$).O sistema deve aplicar uma margem de segurança de 15% para definir a Autonomia Segura ($A_{seg} = A_{max} \times 0.85$).RF02: Motor de Rotas Customizado (OpenStreetMap)Rota Expressa: Prioriza rodovias principais (highway=motorway, highway=trunk) e menor tempo.Rota Sinuosa (Biker Mode): Algoritmo analisa a geometria da via (mudanças angulares de direção por quilômetro) e prioriza estradas secundárias (highway=primary, highway=secondary) com alto índice de curvas, penalizando retas longas.Filtro de Pavimento: Bloqueio estrito ou permissão de vias com a tag surface=unpaved, surface=earth, surface=gravel baseado na escolha do usuário.RF03: Lógica Preditiva de Autonomia ("Modo Tanque Cheio")Na tela de navegação, haverá um botão de ação rápida (fácil clique): "Tanque Cheio ✅".Ao ser acionado, o sistema zera o hodômetro virtual do trajeto ($D_{rodada} = 0$) e inicia o rastreamento via GPS.A cada 500 metros percorridos, o sistema atualiza a Autonomia Restante ($A_{rest} = A_{seg} - D_{rodada}$).Gatilho de Alerta: Quando $A_{rest} \le 40\text{km}$ (ou o equivalente a 20% do tanque), o aplicativo ativa o estado visual de Alerta de Combustível (Modo Reserva).RF04: Busca Dinâmica de POIs (Points of Interest) no TrajetoAo ativar o Alerta de Combustível, ou quando o usuário clica no botão estático "Postos", o app dispara uma requisição geográfica (Overpass API / Banco Local) buscando o marcador amenity=fuel.A busca deve limitar-se a um Buffer Geométrico de 1.000 metros de largura ao longo do vetor da rota restante. Os resultados devem ser ordenados pela distância linear crescente a partir da posição atual do GPS.[ Posição Atual ] ======== ROTA PLANEJADA ========> [ Destino ]

RF05: Desvio Dinâmico de Rota (Waypoint Injection)Se o usuário clicar em um posto sugerido na tela, o sistema exibirá o botão "Desviar para este Posto".O motor de rotas deve recalcular o trajeto adicionando a coordenada do posto como um Waypoint intermediário obrigatório: [Posição Atual] -> [Posto] -> [Destino Final].Ao atingir um raio de 50 metros da coordenada do posto, o sistema assume que o reabastecimento pode ter ocorrido, exibe um pop-up de confirmação de tanque cheio e, após a confirmação ou expiração de 2 minutos, remove o Waypoint, retornando à rota original do ponto onde o usuário está.
